# Анкета заказчика — Sharecode

---

## 1. Идентификация и роли пользователей

### 1.1 [BLOCKER] Как пользователи идентифицируются в комнате?

Это влияет на протокол WebSocket, цвета курсоров и отображение участников.

- (b) Пользователь вводит произвольный никнейм при входе в комнату (без регистрации)

### 1.2 [BLOCKER] Существует ли разграничение ролей внутри комнаты?

Это определяет, нужна ли на бэкенде модель прав.

- (a) Все участники равноправны (любой может редактировать код)

### 1.3 [REFINEMENT] Может ли пользователь сменить отображаемое имя / псевдоним прямо в комнате?

- (b) Да, в любой момент

---

## 2. Управление комнатами

### 2.1 [BLOCKER] Нужна ли защита комнаты паролем?

Без пароля любой, кто знает URL, может войти.

- (a) Нет, достаточно знания URL (закрытость через непредсказуемый ID)

### 2.2 [REFINEMENT] Есть ли ограничение на максимальное число участников в одной комнате?

- (a) Без ограничений

### 2.3 [REFINEMENT] Что происходит с содержимым комнаты после её уничтожения?

- (a) Код удаляется безвозвратно

### 2.4 [REFINEMENT] Может ли создатель комнаты принудительно закрыть её досрочно?

- (b) Да — кнопка «Закрыть комнату»

### 2.5 [REFINEMENT] Существует ли верхняя граница времени жизни комнаты?

- (a) Нет — комната живёт, пока в ней есть хотя бы один участник (плюс 5 мин после)

## 3. Синхронизация и конфликты

### 3.1 [BLOCKER] Какая стратегия разрешения конфликтов при одновременном редактировании?

Это фундаментальный выбор алгоритма и напрямую определяет архитектуру.

- (b) **CRDT** (например, Yjs / Automerge) — без центрального сервера-координатора; проще масштабировать

### 3.2 [REFINEMENT] Нужно ли восстанавливать состояние после кратковременного разрыва WebSocket?

- (a) Да — клиент переподключается и получает актуальное состояние

## 4. Функциональность редактора

### 4.1 [REFINEMENT] Какие языки подсветки синтаксиса включить в первую версию?

В требованиях указан неполный список. Уточните финальный набор:

- (c) Максимальный: все языки, которые поддерживает выбранная библиотека (CodeMirror / Monaco)

### 4.2 [REFINEMENT] Какой язык выбирается по умолчанию при создании новой комнаты?

- (a) Plain text

### 4.3 [REFINEMENT] Нужна ли тема оформления редактора (светлая / тёмная)?

- (c) Переключение light / dark только для текущего окна

### 4.4 [REFINEMENT] Нужны ли дополнительные возможности редактора?

Отметьте всё нужное:

- [+] Автоотступ при переходе на новую строку

### 4.5 [REFINEMENT] Поддерживается ли несколько файлов / вкладок в одной комнате?

- (a) Нет — один редактор, один буфер кода

### 4.6 [REFINEMENT] Есть ли ограничение на объём кода в комнате?

- (b) До 100 КБ

## 5. Дополнительные функции интерфейса

### 5.1 [REFINEMENT] Нужен ли список участников комнаты (sidebar / панель)?

- (b) Да — показывать никнейм и цвет каждого участника

### 5.2 [REFINEMENT] Нужен ли текстовый чат внутри комнаты?

- (a) Нет

### 5.3 [REFINEMENT] Нужна ли возможность запустить код прямо в браузере?

- (a) Нет — редактор только для написания, без выполнения

---

## 6. Инфраструктура и масштабирование

### 6.1 [BLOCKER] Нужна ли горизонтальная масштабируемость бэкенда?

Данные хранятся в памяти. При нескольких инстансах без разделяемого состояния (Redis и т.п.) пользователи одной комнаты должны попадать на один инстанс.

- (a) Нет — один инстанс, горизонтальное масштабирование не нужно

### 6.2 [REFINEMENT] Ожидаемая нагрузка (порядок величин)?

- (a) До 10 одновременных комнат (внутренний инструмент)

### 6.3 [REFINEMENT] Целевая среда развёртывания?

- (a) Один Linux-сервер / VPS (Docker Compose)

---

## 7. Безопасность

### 7.1 [REFINEMENT] Нужно ли ограничение на создание комнат (rate limiting)?

- (b) Да — по IP: не более 100 активных комнат

### 7.2 [REFINEMENT] Нужна ли проверка CORS (разрешённые источники запросов)?

- (a) Нет — открытый CORS (разработка / внутренний инструмент)

---

## 8. UX и доступность

### 8.1 [REFINEMENT] Нужна ли поддержка мобильных устройств?

- (b) Базовая адаптивность (читать можно, удобно редактировать нельзя)

### 8.2 [REFINEMENT] Есть ли требования к доступности (a11y)?

- (a) Нет специальных требований

### 8.3 [REFINEMENT] Есть ли брендинг / дизайн-система?

- (a) Используйте стандартную тему tailwind css
