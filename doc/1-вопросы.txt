Вы являетесь старшим архитектором бэкэнда. Вам необходимо ознакомиться с основными требованиями к сервису ('requirements.md').
Ваша задача — составить анкету для заказчика, которая позволит устранить все пробелы, мешающие разработке уникального дизайна приложения.

1. Сгруппируйте вопросы по разделам, пронумеруйте их («1.1», «1.2», ...) — на них затем будут ссылки в ответах.
2. Для каждого вопроса предложите 2–4 готовых варианта ответа, где это уместно, плюс пункт «другое». Закрытые вопросы лучше, чем открытые.
3. Не задавайте вопросы, на которые уже есть ответы в файле 'requirements.md'.
4. Пометьте каждый вопрос тегом «[BLOCKER]» (продолжить без ответа нельзя) или «[REFINEMENT]» (можно принять разумное значение по умолчанию).
5. В конце добавьте раздел «Default Assumptions» — список решений, которые вы примете сами, если соответствующие вопросы с тегом „[REFINEMENT]“ останутся без ответа.

Сохраните полученный документ как 'questionnaire.md'.
